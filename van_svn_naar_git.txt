1. Committen

Je moet bij git steeds de files die je wilt committen adden. Je kan ook de -a parameter gebruiken. 

	http://book.git-scm.com/3_normal_workflow.html

svn commit -m 'message' = git commit -a -m 'message'

2. Branches

Branches zijn er om makkelijk versies van het project te scheiden. Normaal gesproken is er 1 master branch waar een werkende versie op staat. Wij hebben nog een develop branch waar we tussen de iteraties aan werken. Ik kan je aanraden om zelf lokaal een branch te maken waarin je je eigen aanpassingen maakt bijv: mick branch. 

Op deze manier kun je altijd updates van de server pullen en zorgen dat je eigen veranderingen geen conflicts opleveren voordat je ze commit naar de server. 

In detail
